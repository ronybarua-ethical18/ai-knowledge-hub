## v1.0.0 — Initial Release

AI Knowledge Hub's first release: upload documents into a workspace, and chat with them using RAG grounded in your own content.

### ✨ Features

- **Document upload & indexing** — PDF, DOCX, and TXT (up to 10MB), processed in the background via a BullMQ queue (extract → chunk → embed → index)
- **Workspace-scoped RAG chat** — ask questions grounded in every processed document in your active workspace, with cited source excerpts returned alongside each answer
- **Resilient AI layer** — automatic Gemini model fallback chain, plus an offline extraction fallback if all models are unavailable
- **JWT authentication** — access/refresh tokens, email verification on signup, account lockout after repeated failed logins
- **Workspaces & membership** — auto-created personal workspace on signup, member roles (owner/member/viewer), live document status sidebar
- **Three-pane workspace UI** — documents, chat, and workspace nav in one view, with dark mode

### 🔒 Security hardening

- Added global request validation (`whitelist`/`forbidNonWhitelisted`) — malformed or unexpected input is now rejected at the API boundary instead of reaching the database
- Closed auth guard gaps on `logout`, `me`, and all workspace/member-management endpoints — previously reachable without a valid token
- Removed leakage of internal error details (stack traces, raw DB query text) from API responses in production

### ⚠️ Known limitations

- **Password reset isn't wired up end-to-end yet** — no reset email is sent and there's no reset UI; use `resend-verification` / contact an admin in the meantime
- **No UI to create additional workspaces** — every account gets one workspace at signup; the API supports more, but it's not exposed in the app yet
- Chat retrieval is workspace-wide only — no per-file scoping yet
- Google/LinkedIn OAuth config exists but isn't connected to a working login flow

### 📦 Stack

Next.js · NestJS · PostgreSQL/Prisma · Qdrant · BullMQ/Redis · Google Gemini (chat + embeddings) · TypeScript throughout
